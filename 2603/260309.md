# 260309 TIL

## 면접준비
1. **`전자정부표준프레임워크`**의 **`기반 아키텍처`**는?
    1. `Spring Framework`
2. **`IoC(Inversion of Control)`**란?
    1. `객체의 생성 및 생명주기 제어권`을 개발자가 아닌 `프레임워크`가 갖는 것
3. **`DI(Dependency Injection)`**란?
    1. `객체 간의 의존 관계`를 `외부(컨테이너)에서 주입`해 주는 기술.
4. **`AOP(Aspect Oriented Programming)`**란?
    1. `핵심 로직`과 `공통 기능`을 `분리`하여 `모듈화`하는 관점 지향 프로그래밍
5. **`Spring Bean`**의 **`기본 Scope`**는?
    1. `Singleton(싱글톤)`
6. **`Spring MVC`**의 **`요청 처리 흐름`**은?
    1. `클라이언트` -> `DispatcherServlet` -> `HandlerMapping` -> `Controller` -> `Service` -> `DAO` -> `Controller` -> `ViewResolver` -> `View` -> `클라이언트`
7. **`DispatcherServlet`**의 역할은?
    1. 모든 `클라이언트 요청`을 `한 곳`에서 받아 적절한 `Controller`로 `위임`하는 `Front Controller` 역할
8. **`@Controller`**와 **`@RestController`**의 차이는?
    1. **`@Controller`**는 `View`(JSP 등)를 반환하고, `@RestController`는 `데이터(JSON, XML)`를 반환
9. **`@Autowired`**와 **`@Resource`**의 차이는?
    1. `@Autowired`는 `타입`을 기준으로, `@Resource`는 `이름`을 기준으로 `의존성을 주입`
10. **`Struts 프레임워크`**의 **`컨트롤러 역할 컴포넌트`**는?
    1. `ActionServlet` 및 `Action` 클래스.
11. **`Struts`**와 **`Spring MVC`**의 객체 생성 차이는?
    1. Struts는 요청마다 Action 객체를 생성하지만, Spring MVC는 싱글톤 기반 Controller를 사용.
12. **`트랜잭션 처리`**를 위해 **`선언`**하는 **`Spring 어노테이션`**은?
    1. `@Transactional`
13. **`@Transactional`**의 `기본 전파 속성`은?
    1. `REQUIRED` : 기존 트랜잭션이 있으면 참여, 없으면 새로 생성
14. **`Spring`**의 **`Interceptor`**와 **`Servlet`**의 **`Filter`** 차이는?
    1. `Filter`는 웹 컨테이너 레벨(`DispatcherServlet 진입 전후`)에서 동작하고, `Interceptor`는 Spring 컨텍스트 레벨(`Controller 실행 전후`)에서 동작.
15. **`Servlet`**의 **`생명주기 메서드`**는?
    1. `init()` -> `service()` -> `destroy()`
16. **`JSP`**와 **`Servlet`**의 차이는?
    1. `JSP`는 `HTML` 안에 `자바 코드를 삽입`
    2. `Servlet`은 `자바 코드 내`에 `HTML`을 삽입
    3. `JSP`는 실행 시 `Servlet`으로 변환
17. **`JSP`**의 주요 **`내장 객체 4가지`**는?
    1. `request`
    2. `response`
    3. `session`
    4. `application`
18. **`forward`**와 **`redirect`**의 차이는?
    1. `forward` : `서버 내부 이동`으로 `URL`이 변하지 않고 `request`가 유지
    2. `redirect` : `클라이언트 재요청`으로 `URL`이 변하고 `데이터 초기화`
19. **`Session`**과 **`Cookie`**의 차이는?
    1. `Session`은 `서버` 에 `저장`하여 보안에 상대적으로 강함
    2. `Cookie`는 `클라이언트` 에 저장하여 보안에 취약
20. **`web.xml`**의 역할은?
    1. 웹 애플리케이션 구동에 필요한 `DispatcherServlet`, `Filter`, `Listener` 등의 `초기 설정` 정의
21. **`데이터베이스 커넥션 풀(DBCP)`**의 사용 목적은?
    1. `DB 커넥션`을 `미리 생성`해두고 `재사용`하여 데이터베이스 `연결 소요 시간`과 `자원`을 `절약`해 `성능`을 향상.
22. **`전자정부프레임워크 공통컴포넌트`**의 목적은?
    1. `자주 사용되는 기능`(로그인, 게시판, 권한관리 등)을 `미리 구현`해 두어 `개발 생산성`과 `재사용성`을 높임.
23. **`@Service`, `@Repository`, `@Component`**의 차이는?
    1. 모두 빈 등록용 어노테이션이나, 역할에 따라 `비즈니스 로직(@Service)`, `데이터 접근(@Repository)`, `일반 컴포넌트(@Component)`로 구분함.
24. **`mybatis-spring`** 모듈의 역할은?
    1. `MyBatis`와 `Spring 트랜잭션`을 연동하고 `Mapper 인스턴스`를 `Spring 빈`으로 주입할 수 있게 함.
25. `전자정부표준프레임워크`의 `실행환경 구성 요소 4가지`는?
    1. `화면처리`
    2. `업무처리`
    3. `데이터처리`
    4. `공통기반 계층`
