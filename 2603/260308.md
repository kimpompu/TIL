# 260308 TIL

- **`객체지향 프로그래밍(OOP)`**의 4가지 특징은?
    - `캡슐화`
    - `상속`
    - `다형성`
    - `추상화`
- **`Interface`**와 **`Abstract Class`**의 차이는?
    - `인터페이스`는 `다중 구현이 가능`하고 일반 변수를 가질 수 없음
    - `추상 클래스`는 `단일 상속`만 가능하며 `일반 메서드와 멤버 변수`를 가질 수 있음
- **`Overloading`**과 **`Overriding`**의 차이는?
    - `오버로딩`은 같은 이름의 메서드를 `매개변수 타입/개수`를 다르게 하여 여러 개 정의
    - `오버라이딩`은 `상위 클래스`의 메서드를 `하위 클래스`에서 재정의
- **`String`, `StringBuilder`, `StringBuffer`**의 차이는?
    - `String`은 불변
    - `StringBuilder`와 `StringBuffer`는 가변
    - `StringBuffer`는 동기화를 지원하고 `StringBuilder`는 지원하지 않음
- **`JVM`** 메모리 구조는?
    - `Method Area`
    - `Heap`
    - `Stack`
    - `PC Register`
    - `Native Method Stack`
- **`Garbage Collection`**의 대상이 되는 메모리 영역은?
    - `Heap`
- **`Checked Exception`**과 **`Unchecked Exception`**의 차이는?
    - `Checked Exception`은 `컴파일` 시점에 `예외 처리를 강제`
    - `Unchecked Exception`(`RuntimeException` 상속)은 예외 처리를 강제하지 않음
- **`List`, `Set`, `Map`** 컬렉션의 차이는?
    - `List`는 **순서가 있고 중복 허용**
    - `Set`은 **순서가 없고 중복 불가**
    - `Map`은 **Key-Value 쌍**이며 **Key 중복 불가**, **Value 중복 가능**.
- **`HashMap`**과 **`HashTable`**의 차이는?
    - `HashTable`은 `동기화를 지원`하고 `Null` 비허용
    - `HashMap`은  `동기화` 미지원, `Null Key/Value를 허용`
- Q: **`접근 제어자`**의 종류와 범위는?
    - `public` : 체
    - `protected` : 동일 패키지+자식 클래스
    - `default` : 동일 패키지
    - `private` : 동일 클래스
- **`static`** 키워드의 의미는?
    - `클래스 레벨의 변수/메서드로 선언`되어, `인스턴스 생성 없이 클래스 이름으로 접근 가능`하며 `Method Area에 적재`됨.
- **`final`** 키워드의 용도는?
    - `변수`에 사용 시 `상수화`
    - `메서드`에 사용 시 `오버라이딩 불가`
    - `클래스`에 사용 시 `상속 불가`
- **`제네릭(Generic)`**의 장점은?
    - `컴파일` 시점에 `타입을 체크`하여 `타입 안정성을 보장`하고, `불필요한 타입 캐스팅`을 `생략`할 수 있음
- **`자바`**에서 **`멀티스레드 구현 방법`** 두 가지는?
    - `Thread` 클래스 상속
    - `Runnable` 인터페이스 구현
- **`동기화(Synchronization)`**의 목적은?
    - `멀티스레드` 환경에서 `공유 자원`에 대한 `동시 접근을 제어`하여 `데이터의 일관성을 유지`
- `Java 8`에 추가된 주요 기능은?
    - 람다
    - 스트림 API
    - Optional
    - 인터페이스 Default 메서드
- **`싱글톤(Singleton) 패턴`**의 목적은?
    - `애플리케이션 전체`에서 특정 클래스의 `인스턴스가 단 하나만 생성`되도록 보장
- **`MVC 패턴`**의 구성 요소와 역할은?
    - `Model` : 데이터 및 로직
    - `View` : 사용자 화면
    - `Controller` : 요청 처리 및 흐름 제어
- **`DTO`**와 **`VO`**의 차이는?
    - `DTO`는 `계층 간 데이터 교환`을 위한 `가변` 객체
    - `VO`는 `값 자체`를 표현하는 `불변` 객체
- **`Wrapper Class`**란?
    - `기본 자료형(Primitive Type)`을 객체 형태로 다루기 위해 사용하는 클래스
        - ex) Integer, Double
- **`equals()`**와 **`==`**의 차이는?
    - `==`는 객체의 `물리적 주소값`을 비교
    - `equals()`는 객체의 `논리적 내부 값`을 비교.
- **`직렬화(Serialization)`**란?
    - `자바 객체`를 네트워크 `전송`이나 `파일 저장`을 위해 `바이트 스트림 형태`로 `변환`하는 과정.
- **`리플렉션(Reflection)`**이란?
    - `런타임`에 `클래스의 정보`(메서드, 변수 등)를 `분석`하고 `접근`할 수 있게 해주는 API
- `배열 내 중복 요소`를 `제거`하기 가장 좋은 `자료구조`는?
    - `HashSet`
- `LIFO`, `FIFO` 구조를 가진 `자료구조`는 각각 무엇인가?
    - `LIFO`는 `Stack`
    - `FIFO`는 `Queue`
