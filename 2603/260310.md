# 260310 TIL

## React 

### 1. `JSX`
* **개념 :** `JavaScript` 내에서 `HTML`과 유사한 구문을 작성할 수 있게 해주는 React의 확장 문법
* **특징 :** 브라우저 실행 전 `Babel`과 같은 컴파일러에 의해 일반 JavaScript 객체(`React.createElement`)로 변환
* **규칙 :** 반드시 `하나의 부모 요소`로 감싸야 하며, `JavaScript 표현식`은 중괄호 `{}` 안에 작성

### 2. `Component`
* **개념 :** `UI를 구성`하는 독립적이고 `재사용 가능`한 코드 블록
* **종류:** 
    * **클래스형 컴포넌트 :** `React.Component`를 상속받으며, `생명주기 메서드`와 내부 상태(`State`)를 가짐
  * **함수형 컴포넌트 :** 순수 JavaScript 함수로 작성되며, `React 16.8` 이후 `Hooks`가 도입되면서 현재 표준으로 사용

### 3. Props와 State
* **Props(Properties) :** `부모 컴포넌트`에서 `자식 컴포넌트`로 전달되는 `읽기 전용 데이터`. 단방향 데이터 바인딩을 형성
* **State:** 컴포넌트 `내부`에서 선언되고 관리되는 동적 데이터. `State`가 `변경`되면 `컴포넌트`는 `해당 변경 사항을 반영하기 위해 리렌더링`됨

### 4. React Hooks
* **`useState` :** 컴포넌트 `내부`에 `상태를 선언`하고 `업데이트`하는 함수
* **`useEffect` :** `렌더링 이후` 발생하는 부수 효과(`Side Effect` : API 호출, DOM 조작, 타이머 설정 등)를 처리.
* **`useRef` :** `DOM 요소`에 `직접 접근`하거나, 렌더링에 영향을 주지 않는 `변경 가능한 변수`를 유지할 때 사용.
