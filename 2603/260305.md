## 구현
- 자바에서 `Queue`는 `인터페이스`라 `new Queue()`로 직접 구현할 수 없음
- `LinkedList` 사용해 구현

```java
    import java.util.LinkedList;
    import java.util.Queue;

    Queue<Integer> queue = new LinkedList<>();

    queue.offer(1);
    queue.offer(2); 
    queue.offer(3); 

    System.out.println(queue.poll()); // 1 반환 후 제거 (남은 것: [2, 3])
    System.out.println(queue.peek()); // 2 최상단 데이터 확인 (남은 것: [2, 3])
    System.out.println(queue.isEmpty()); // 큐가 비었는지 확인 (false)
```

## 문제 유형
1. 순서대로 처리
- 특징 : 작업들이 순차적으로 들어오고, 특정 조건이 만족될 때까지 뒤의 작업이 대기하는 경우
    ex) 프린터 인쇄 대기열, CPU 스케줄링

2. 너비 우선 탐색
- 특징 :탐색에서 **가장 가까운 곳부터** 차례대로 방문할 때 사용

3. 시간/거리 시뮬레이션
- 특징 :일정 시간 동안만 큐에 머물다가 나가는 상황을 시뮬레이션

## Queue 메소드
| 메소드 | 기능 | 비고 |
| :--- | :--- | :--- |
| `offer(e)` | 데이터 삽입 | 큐가 꽉 찼을 때 예외를 던지지 않음 |
| `poll()` | 맨 앞 데이터 꺼내기 | 큐가 비어있으면 `null` 반환 |
| `peek()` | 맨 앞 데이터 확인 | 제거하지 않고 값만 읽음 |
| `size()` | 큐의 길이 확인 |  |
