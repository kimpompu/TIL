# 260307 TIL

## 해시
- `HashMap`과 `HashSet`
- `key`-`value` 구조
- 순서 비보장
    - 순서가 필요할 경우 `LinkHashMap` 사용
- `Null` 허용

## 출제 유형
1. 존재 여부 확인
    - 특정 원소가 집합 내에 있는지 확인해야 할 때
        ex) 전화번호부 접두어 찾기, 완주하지 못한 선수 찾기
2. 개수 세기
    - 각 항목이 몇 번 등장했는지 카운팅
        ex) 가장 많이 등장한 단어 뽑기, 의상 조합 수 계산
3. 중복 제거
    - 데이터에서 중복을 없애고 유일한 값만 남겨야 할 때

## 풀이법
1. 해시에 데이터 추가
2. 반복 처리
3. 예외처리 및 종료 확인


### 메소드
| 클래스 | 메소드 | 설명 | 시간 복잡도 |
| :--- | :--- | :--- | :--- |
| `HashMap` | `put(k, v)` | 키와 값을 저장 | $O(1)$ |
| `HashMap` | `get(k)` | 키에 해당하는 값 반환 | $O(1)$ |
| `HashMap` | `getOrDefault(k, d)` | 키가 없으면 기본값(`d`) 반환 | $O(1)$ |
| `HashMap` | `keySet()` | 모든 키를 `Set` 형태로 반환 | $O(n)$ |
| `HashSet` | `add(e)` | 원소 추가 (중복 시 무시) | $O(1)$ |
| `HashSet` | `contains(e)` | 원소 존재 여부 확인 | $O(1)$ |
