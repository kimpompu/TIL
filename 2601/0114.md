## 단기, 중기, 장기 스케쥴러에 대해 설명해 주세요.

### 장기 스케줄러
- 디스크 내의 작업 풀에서 어떤 프로세스를 메모리에 적재할지 결정
- 시스템 내의 프로세스 개수(다중 프로그래밍의 정도)를 조절하는 역할

### 단기 스케줄러
- 메모리 내의 준비 큐에 있는 프로세스 중 어떤 프로세스에 중앙처리장치를 할당할지 결정
- 매우 빈번하게 발생하며 속도가 매우 빨라야 함

### 중기 스케줄러
- 메모리에 적재된 프로세스 수가 너무 많아 성능이 저하될 때, 일부 프로세스를 메모리에서 디스크의 스왑 영역으로 내보내는 역할
- 다중 프로그래밍의 정도를 동적으로 완화함

## 현대 운영체제에는 단기, 중기, 장기 스케쥴러를 모두 사용하고 있나요?

### 현대 운영체제의 특징
현대적 시분할 운영체제(윈도우, 리눅스 등)에서는 장기 스케줄러를 거의 사용하지 않음. 사용자가 프로그램을 실행하면 바로 메모리에 적재되어 준비 상태가 됨. 

### 사용 현황
* **단기 스케줄러 :** 중앙처리장치 할당을 위해 사용
* **중기 스케줄러 :** 메모리 부족 상황에 대비하여 모든 현대 운영체제에서 스와핑 기법으로 사용
* **장기 스케줄러 :** 과거 일괄 처리 시스템에서 주로 사용되었으며, 현재는 시분할 시스템의 특성상 거의 존재하지 않음

## 프로세스의 스케쥴링 상태에 대해 설명해 주세요.

### 주요 상태 종류
1. **생성 :** 프로세스가 이제 막 생성되어 운영체제에 등록
2. **준비 :** 중앙처리장치를 점유하기 위해 메모리에서 대기
3. **실행 :** 중앙처리장치를 할당받아 명령어 수행
4. **대기 :** 입출력 작업이나 특정 이벤트가 완료되기를 기다리며 중앙처리장치를 사용할 수 없는 상태
5. **종료 :** 실행 완료 후 자원이 회수
6. **지연 :** 중기 스케줄러에 의해 메모리에서 쫓겨나 디스크로 스왑 아웃

## 선점형/비선점형에서 존재할 수 없는 상태가 있을까요?

### 상태의 존재 유무 분석
모든 스케줄링 방식에서 준비, 실행, 대기 등의 기본적인 상태는 모두 존재함. 하지만 상태 간의 '전이(이동)'에서 차이가 발생함.

### 존재하지 않는 전이 과정
* **비선점형 스케줄러 :** 실행 상태에서 준비 상태로 강제 전환되는 전이가 존재하지 않음. 프로세스가 스스로 종료되거나 입출력을 위해 대기 상태로 가지 않는 한 중앙처리장치를 빼앗기지 않음.
* **선점형 스케줄러 :** 할당 시간이 종료되거나 우선순위가 높은 프로세스가 등장하면 실행 중인 프로세스가 준비 상태로 강제 복귀하는 전이가 존재함.

## 메모리가 부족할 경우, 프로세스는 어떠한 상태로 변화할까요?

### 상태 변화 과정
중기 스케줄러가 개입하여 메모리에 여유 공간을 확보하기 위해 실행 빈도가 낮거나 대기 시간이 긴 프로세스를 선택함.

### 결과 상태
* **지연 상태 :** 메모리 공간을 반납하고 디스크로 이동
* **구체적 구분 :** 대기 중이던 프로세스는 '지연 대기' 상태로, 준비 큐에 있던 프로세스는 '지연 준비' 상태로 변화. 메모리 공간이 확보되면 다시 준비 상태로 돌아와 실행을 기다림.
