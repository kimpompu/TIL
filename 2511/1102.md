## Java의 Generic에 대해 설명하고, Type Erasure란 무엇인가요?

### 제네릭(Generic)
- `List<String>`
- 클래스나 메소드가 다룰 데이터 타입을 컴파일 시점에 미리 지정할 수 있게 해 주는 기능

### 타입 소거(Type Erasure)
- 자바 컴파일러가 제네릭이 적용된 코드를 바이트코드로 변환할 때, 제네릭 타입 정보(예: `<String>`, `<Integer>`)를 제거하는 과정
- 런타임(JVM)에서는 제네릭 타입 정보가 존재하지 않음. 예를 들어, List<String>과 List<Integer>는 런타임에 모두 단순한 Raw Type인 List로 간주  

1. 원리
    - 비한정 타입 매개변수 (<T>): Object 타입으로 대체

    - 한정 타입 매개변수 (<T extends Number>): 경계(Number) 타입으로 대체

    - 타입 안전성을 유지하기 위해 필요하다면 컴파일러가 적절한 형 변환(Type Cast) 코드를 삽입

 

## Comparable과 Comparator의 차이점을 설명해주세요.

### `Comparable<T>`
- 정의하는 클래스 내부에 자기 자신끼리의 자연스러운 정렬 기준(natural ordering)을 정의할 때 사용
- `java.lang.Comparable<T>`패키지에 속해 있으며, 메소드 `int compareTo(T o)` 하나를 구현
- 한 클래스에 대해 하나의 정렬 기준만 정의
- 클래스 내부에 정렬 로직이 들어가므로 그 클래스 소스를 수정할 수 있어야 적용 가능

### `Comparator<T>`
- 외부에서 다양한 정렬 기준(custom ordering)을 정의해야 할 때 사용
- `java.util.Comparator<T>` 패키지에 속하며, 메소드 `int compare(T o1, T o2)`를 구현(또는 람다식)
- 클래스 소스를 수정할 수 없는 외부 라이브러리 객체 또는 동일 객체를 여러 방식으로 정렬해야 할 경우

| 항목        | `Comparable<T>`        | `Comparator<T>`       |
| --------- | ---------------------- | --------------------- |
| 목적        | 객체 자체에 자연스러운 정렬 기준을 정의 | 외부에서 다양한 정렬 방법을 정의    |
| 메소드       | `compareTo(T o)`       | `compare(T o1, T o2)` |
| 위치        | 클래스 내부(자기 클래스가 구현)     | 별도 클래스 혹은 람다 표현으로 구현  |
| 정렬 기준 수   | 보통 하나 (단일 정렬 기준)       | 여러 개 가능 (여러 기준)       |
| 클래스 수정 여부 | 클래스 수정 필요              | 클래스 수정 없이도 구현 가능      |
| 구현 패키지    | `java.lang`            | `java.util`           |

- 클래스가 스스로 정렬될 “자연스러운 기준”이 있고, 해당 클래스를 주로 한 방식으로 정렬할 계획이라면 Comparable

- 여러 정렬 기준이 필요하거나, 클래스 코드를 수정할 수 없으며 외부에서 정렬 로직을 정의하고 싶다면 Comparator가 유리
