## Spring Framework의 기본 개념과 주요 특징에 대해 설명해주세요.

### Spring Framework
- 자바 플랫폼 위에서 엔터프라이즈 애플리케이션을 보다 가볍고 유연하게 개발하도록 돕는 오픈소스 프레임워크

### 특징
- 제어의 역전(IoC: Inversion of Control)
- 의존성 주입(DI: Dependency Injection)
- 개발자가 객체를 직접 생성하고 의존 관계를 관리하기보다는 프레임워크(컨테이너)가 객체 생성 및 의존성 주입을 관리

| 특징         | 설명                                                                                                                    |
| ---------- | --------------------------------------------------------------------------------------------------------------------- |
| IoC / DI   | Spring IoC 컨테이너가 객체 생성과 생명주기, 의존성 주입을 관리함으로써 코드의 결합도를 낮추고 테스트 가능성을 높입니다. ([Java Guides][1])                           |
| AOP 지원     | 로깅, 보안, 트랜잭션 등 공통 관심사를 분리해서 비즈니스 로직과 독립적으로 관리할 수 있게 합니다. ([devkuma.com][2])                                           |
| POJO 기반 개발 | Spring은 특정 인터페이스나 클래스를 상속해야만 사용하는 방식이 아닌, 일반 자바 객체(POJO)를 활용하여 개발할 수 있게 설계되어 있습니다. ([springjava.com][3])              |
| 모듈화 / 유연성  | 필요에 따라 Core Container, 데이터 접근(Data Access/Integration), 웹(Web/MVC), 테스트(Test) 등의 모듈을 별도로 선택해서 사용할 수 있습니다. ([Home][4]) |
| 트랜잭션 관리    | 로컬 트랜잭션부터 글로벌(JTA)까지 일관된 추상화를 제공하여 데이터베이스/ORM 등의 복잡한 트랜잭션 처리도 쉽게 합니다. ([Java Guides][5])                              |
| 테스트 지원     | Spring ApplicationContext 기반 테스트 지원, Mock 객체 지원 등 단위시험 및 통합시험에 유리한 구조를 제공합니다. ([Home][4])                             |

## Spring Boot와 전통적 Spring Framework의 차이점은 무엇인가요?

| 항목                | Spring Framework(전통적)                                                                | Spring Boot                                                                                                                 |
| ----------------- | ------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------- |
| 설정(Configuration) | XML 기반 또는 Java Config를 통해 직접 빈(bean), 컨테이너, 서블릿 등을 상세히 구성해야 합니다. ([kodejava.org][1]) | 자동구성(auto-configuration)을 통해 개발자가 최소한의 설정만으로도 작동하게 됩니다. “컨벤션 대신 설정(Convention over Configuration)” 원칙이 적용됩니다. ([Medium][2]) |
| 서버 배포             | 외부 웹서버(Tomcat, WebLogic 등)에 WAR 형태로 배포하는 방식이 일반적입니다. ([GeeksforGeeks][3])            | 내장(embedded) 웹서버(Tomcat, Jetty, Undertow 등)를 포함해 독립 실행 가능한 JAR 형태로 배포가 가능합니다. ([seoinfotech.com][4])                        |
| 의존성 관리 및 스타터      | 개별 라이브러리 의존성을 직접 관리해야 하며 설정이 번거롭습니다. ([XcelTec][5])                                  | “스타터(Starter)” POM/Gradle 설정을 제공해 자주 사용하는 라이브러리를 묶음으로 관리해주며 버전 충돌을 줄입니다. ([Medium][2])                                      |
| 학습 곡선 및 생산성       | 유연하고 강력하지만 초기 설정이 복잡하고 학습 곡선이 가파를 수 있습니다. ([KapreSoft][6])                           | 빠른 개발이 가능하며 설정량이 적어 새 프로젝트 시작 시 생산성이 높습니다. ([seoinfotech.com][4])                                                           |
| 프로덕션 준비 기능        | 모니터링, 메트릭, 힙/쓰레드 체크 등은 별도 설정이 필요합니다. ([Java Guides][7])                              | 프로덕션 수준에서 필요한 기능(Actuator, Health 체크, 메트릭 등)이 기본 제공됩니다. ([codingeasypeasy.com][8])                                          |
